## 1. Purpose

The `track-manifest.json` is a public-facing governance artifact. Its purpose is to provide a standardized, machine-readable summary of a NINE² niche track, allowing potential cohort members, institutional partners, and regulators to understand the track's purpose, governance posture, and operational parameters at a glance. This file is a cornerstone of the **Niche Discoverability** phase, making governance transparent and verifiable.

## 2. Schema and Example

The manifest is a simple JSON file hosted at a predictable public URL (e.g., `https://api.nine2.ai/v1/tracks/vnt-03/manifest.json`).

```json
{
  "schema_version": "1.0",
  "track_id": "VNT-03",
  "name": "Nurse Venture Builder",
  "clinical_vertical": "venture",
  "icp": "Entrepreneurial nurses launching AI-assisted healthcare products, consulting practices, or advocacy organizations.",
  "anchor_workflow": "Business model validation -> pitch deck assembly -> stakeholder communication system",
  "governance_summary": {
    "naio_steward_assigned": true,
    "edena_default_tier": "yellow",
    "human_gate_on_yellow": true,
    "red_tier_blocked": true,
    "ai_disclosure_enforced": true
  },
  "endpoints": {
    "agent_registry": "https://api.nine2.ai/v1/tracks/vnt-03/agent-registry.json",
    "governance_log_summary": "https://api.nine2.ai/v1/tracks/vnt-03/audit/summary"
  },
  "last_updated": "2026-03-05T14:00:00Z"
}
```

## 3. Field Descriptions

| Field | Description |
| :--- | :--- |
| `schema_version` | The version of the manifest schema itself. |
| `track_id` | The unique, immutable ID of the track (from `cohort-config.yaml`). |
| `name` | The human-readable name of the track. |
| `clinical_vertical` | The primary domain of the track. |
| `icp` | The Ideal Cohort Profile, describing the target nurse for this track. |
| `anchor_workflow` | The primary workflow the track is designed to address. |
| `governance_summary` | A high-level summary of the track's key governance settings, allowing for a quick risk assessment. |
| `endpoints` | A list of public URLs where other governance artifacts and data can be found. This is crucial for programmatic discovery. |
| `last_updated` | An ISO 8601 timestamp indicating the last time the manifest was updated. |

## 4. Generation and Hosting

- **Generation:** The `track-manifest.json` file is automatically generated by the `fork-track.sh` script and updated by the CI/CD pipeline whenever the `cohort-config.yaml` is changed.
- **Hosting:** The file is hosted on a public, stable, and versioned API endpoint. This ensures that third parties can rely on the location and format of the manifest for their own verification tools.

This manifest transforms governance from a set of internal policies into a public, verifiable declaration of intent and operation, which is essential for building trust at scale.
